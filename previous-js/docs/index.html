Introduction

Welcome to the Previous.js documentation!

What is Previous.js?

Previous.js is a framework for building web applications.

With Previous.js, you can build user interfaces using HTML + jQuery + CSS. Then, Previous.js provides additional structure, features, and optimizations for your application.

Under the hood, Previous.js also abstracts and automatically configures tooling for you, like bundling, compiling, and more. This allows you to focus on building your application instead of spending time setting up tooling.

Whether you're an individual developer or part of a larger team, Previous.js can help you build interactive, dynamic, and fast web applications.

Main Features

Some of the main Previous.js features include:

Feature Description
Routing Create routes using the file system. Navigate with usual HTML links and instantly stream UI from the server with jQuery Ajax.
Rendering Render contents using pages and nested layouts within your app routing structure and load resources dynamically only when necessary.
Data Simplified data fetching synchronized with your app views in both client and server side.
Styling Style your application with pure native CSS including any libraries you want to use with dynamic loading.
APIs Make API endpoints to securely connect with third party services and consume from your frontend.

How to Use These Docs

The sections and pages of the docs are organized sequentially, from basic to advanced, so you can follow them step-by-step when building your Previous.js application. However, you can read them in any order or skip to the pages that apply to your use case.

On the right side of the page, you'll see a table of contents that makes it easier to navigate between sections of a page. The breadcrumbs at the top of the page will also indicate whether you're viewing App Router docs or Pages Router docs.

To get started, checkout the Installation. If you're new to HTML5 or Server Components, we recommend reading Learn HTML5 before continuing.
